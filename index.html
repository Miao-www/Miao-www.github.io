Now,let’s look at some parameters that have important impact on the security of the scheme. We define p as plaintext modulus, q as ciphertext modulus, and as scalling factor.The fact is that bigger q means larger initial noise budget in a freshly encrypted ciphertext,this is good，because we can do more operations before refreshing the ciphertext with bootstrapping which consumes lots of time, however, it also means lower security with bigger q , in principle, we can take q to be any integer, as long as it is not too large to cause security problems.Here is a small example to help us better understand the problem.
